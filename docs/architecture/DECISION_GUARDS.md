# Decision Guards

## ADR обязателен, когда изменение затрагивает

- Структуру монорепозитория и границы пакетов.
- Публичный контракт `@genora/ui` или `@genora/database`.
- Стратегию аутентификации и управление сессиями.
- Платёжный поток и обработку webhooks.
- Модель данных (схема Prisma) с влиянием на существующие данные.
- Добавление или замену ключевой зависимости стека.

## Пороги для ревью

- Регресс Core Web Vitals на ключевых страницах.
- Изменение семантики кеширования/ревалидации данных.
- Изменение границ между `app/`, `features/` и пакетами.
- Любое нарушение `standards/CODE_STANDARD.md`.

## Процедура

- Решение фиксируется по `templates/ADR_TEMPLATE.md`.
- Запись добавляется в `decisions/ADR-INDEX.md`.
