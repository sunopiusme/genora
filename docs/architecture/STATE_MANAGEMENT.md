# State Management

## Разделение ответственности

- Клиентское состояние UI -> Zustand.
- Серверные данные -> TanStack Query.

Эти два слоя не смешиваются: серверные данные не дублируются в Zustand.

## Zustand

Глобальные клиентские сторы:

- `cartStore` — корзина.
- `userStore` — клиентское представление пользователя.
- `uiStore` — состояние интерфейса (модалки, тема, панели).

## TanStack Query

Серверные данные с кешем и инвалидизацией:

- заказы,
- продукты,
- профиль и подписки.

## Правила

- Источник истины серверных данных — сервер; клиент кеширует, а не владеет.
- Инвалидизация после мутаций выполняется явно.
- Сторы Zustand — минимальные и сериализуемые там, где это нужно.
