# Routing Map

## Если задача про...

- качество и стиль кода -> `standards/CODE_STANDARD.md`
- продуктовый оффер/позиционирование -> `foundations/*`
- структуру репозитория/монорепо/пакеты -> `architecture/REPO_STRUCTURE.md`, `architecture/MONOREPO.md`
- собственную UI библиотеку и компоненты -> `architecture/UI_LIBRARY.md`, `standards/UI_STANDARDS.md`
- структуру фичи -> `architecture/FEATURE_STRUCTURE.md`
- данные, Server Actions, Prisma -> `architecture/DATA_LAYER.md`
- состояние клиента и серверные данные -> `architecture/STATE_MANAGEMENT.md`
- аутентификацию -> `architecture/AUTH.md`
- реализацию Next.js/TypeScript -> `standards/NEXTJS_STANDARDS.md`, `standards/TYPESCRIPT_STANDARDS.md`
- технический стек и версии -> `stack/TECH_STACK.md`
- спорное техническое решение -> `decisions/*`
- окружения, конфиги, релизные риски -> `operations/*`

## Definition of Ready для разработки

Перед началом реализации должны быть заполнены:

- цель и критерий приёмки задачи,
- архитектурное ограничение (где живёт код, какие пакеты затрагиваются),
- путь проверки (как и где результат валидируется),
- соответствие `standards/CODE_STANDARD.md`.
