# Manifest

## Назначение

Определяет роль документации как «operating system» для разработки Genora Pro.

## Принципы

- Один источник истины на каждый тип решения.
- Документ сначала задаёт контекст, потом правило, потом критерий готовности.
- Если правило конфликтует с реализацией, инициируется ADR.

## Приоритет источников

1. `standards/CODE_STANDARD.md` — главный стандарт кода, выше всего.
2. `decisions/*` (утверждённые ADR)
3. `architecture/*`
4. `standards/*` (кроме CODE_STANDARD), `stack/*`, `playbooks/*`
5. `foundations/*`
6. `templates/*` (не источник истины, а форма)

## Главный документ

`standards/CODE_STANDARD.md` фиксирует правило читаемости кода и запрет комментариев.
Он обязателен к прочтению до написания любого кода и не имеет исключений.
