# TypeScript Standards

## Назначение

Правила типизации для всех пакетов монорепозитория.

## Обязательные правила

- `strict: true`, общий базовый `tsconfig.base.json`.
- Без `any`. Неизвестное — `unknown` с явной проверкой.
- Без `@ts-ignore`; `@ts-expect-error` только с обоснованием.
- Валидация внешних данных через Zod, затем вывод типа из схемы.
- Публичные контракты пакетов экспортируются явно через `index.ts`.

## Контракты данных

- Схема Zod — единый источник истины для формы и API одной фичи.
- Тип выводится из схемы (`z.infer`), а не дублируется вручную.
- Доменные типы фичи живут в `features/<feature>/types.ts`.
