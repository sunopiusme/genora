# UI Standards

## Назначение

Правила для собственной UI библиотеки `@genora/ui` и её использования в приложении.

## Принципы

- Доступность на основе Radix primitives; визуальный слой полностью свой.
- Токены дизайна берутся из `@genora/design-tokens`, без «магических» значений в компонентах.
- Компонент не содержит бизнес-логики и не обращается к данным.
- Варианты стилей описываются декларативно (class-variance-authority), без ветвлений в разметке.

## Высота поверхностей (elevation)

Правило: чем темнее фон под элементом, тем светлее его поверхность.

- Обычные меню и поповеры поверх контента (chat, композер и т.д.) используют
  `--color-surface-elevated` (`neutral-800`): контент лежит на `--color-surface`
  / тёмных панелях, где этой разницы достаточно, а тень (`--shadow-lg`)
  дорисовывает глубину.
- Элементы, всплывающие над чисто чёрным (#000) фоном сайдбара — меню профиля,
  и подобные оверлеи — используют `--color-surface-raised` (`neutral-750`).
  Причина: на чистом чёрном тени физически не видны (тень темнее фона быть
  не может), поэтому единственный способ передать «высоту» — светлота.
  Более светлый фон однозначно отделяет плавающий слой от рейла.
- Hover-состояния в сайдбаре: пункты навигации используют `--color-surface`;
  кнопка toggle — `--color-surface-elevated` (на ступень светлее), так как
  это control, который появляется поверх логотипа и должен читаться мгновенно.
- Бордеры и разделители внутри «raised»-поверхностей задаются полупрозрачным
  белым (`rgb(255 255 255 / 0.06–0.08)`), а не `--color-border`: токен бордера
  (`neutral-800`) темнее самой поверхности и выглядит как грязная тень.
- Текст тултипов — жирный белый (`--neutral-0`, `--weight-semibold`):
  тултипы мелкие и живут на тёмном фоне, максимальный контраст обязателен.

## Слои библиотеки

- Primitives — низкоуровневые элементы: Button, Input, Checkbox, Dialog, Tooltip и т.д.
- Components — бизнес-UI: PricingCard, ProductCard, SubscriptionCard, AIProviderBadge, OrderStatus.
- Blocks — готовые секции: HeroSection, PricingSection, CheckoutBlock, DashboardSidebar.

## Документация компонентов

- Каждый компонент имеет Storybook story.
- Спецификация компонента фиксируется по `templates/COMPONENT_SPEC_TEMPLATE.md`.
