# ADR-0005: Разделение состояния — Zustand и TanStack Query

- Status: Accepted
- Date: 2026-06-27

## Context

Нужно чёткое разделение клиентского состояния UI и серверных данных, чтобы избежать
дублирования и рассинхронизации.

## Decision

Клиентское состояние UI — в Zustand. Серверные данные — в TanStack Query. Серверные
данные не дублируются в Zustand.

## Consequences

- Понятные границы ответственности слоёв состояния.
- Кеш и инвалидизация серверных данных в одном месте.
- Требуется дисциплина: не копировать серверные данные в клиентские сторы.

## Alternatives Considered

- Единый глобальный стор для всего: смешивает источники истины и усложняет инвалидизацию.
