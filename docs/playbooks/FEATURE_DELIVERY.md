# Feature Delivery Playbook

## Назначение

Стандартный конвейер внедрения функциональности в Genora Pro.

## Ready checklist

- Цель и критерий приёмки зафиксированы.
- Определены затрагиваемые фичи и пакеты.
- Понятна стратегия данных и кеширования.

## Build checklist

- Код размещён по `architecture/FEATURE_STRUCTURE.md`.
- Вход мутаций валидируется Zod-схемой.
- UI собран из `@genora/ui`, без бизнес-логики в компонентах.
- Соблюдён `standards/CODE_STANDARD.md` (чистый код, без комментариев).

## Review checklist

- `pnpm lint` и `pnpm typecheck` проходят.
- Нет дублирования серверных данных в Zustand.
- Изменения с архитектурным влиянием сопровождаются ADR.

## Release checklist

- Core Web Vitals в пределах порогов.
- Платёжный поток проверен в тестовом режиме (если затронут).
- Переменные окружения обновлены в `.env.example` при необходимости.
